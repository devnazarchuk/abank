# День 4: Налаштування Аутентифікації (Clerk)

**Дата:** 26.01.2026
**Мета:** Підключити та налаштувати сервіс Clerk для безпечної реєстрації та входу користувачів.

---

## 1. Завдання на день

- [x] **Аналіз поточного стану:**
  - Проект `AI-LMS` вже має інтеграцію з Clerk (`@clerk/nextjs`).
  - Сторінки `/sign-in` та `/sign-up` вже створені в папці `app/(auth)`.

- [ ] **Отримання API ключів (Завдання для користувача):**
  - [ ] Зареєструватися на [clerk.com](https://clerk.com).
  - [ ] Створити новий додаток "AI LMS".
  - [ ] Скопіювати `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` та `CLERK_SECRET_KEY`.
  - [ ] Вставити ключі в `.env.local`.

- [ ] **Налаштування Redirects:**
  - [ ] Переконатися, що в `middleware.ts` (або `middleware.js`) налаштовані правильні публічні маршрути.
  - [ ] Перевірити змінні в `.env.local`:
    ```env
    NEXT_PUBLIC_CLERK_SIGN_IN_URL=/sign-in
    NEXT_PUBLIC_CLERK_SIGN_UP_URL=/sign-up
    NEXT_PUBLIC_CLERK_AFTER_SIGN_IN_URL=/
    NEXT_PUBLIC_CLERK_AFTER_SIGN_UP_URL=/
    ```

- [ ] **Перевірка роботи:**
  - [ ] Запустити сервер `npm run dev`.
  - [ ] Спробувати зареєструватися.
  - [ ] Перевірити, чи з'явився користувач в панелі Clerk.

## 2. Результат дня

- Працююча система входу/реєстрації через Google/Email.
- Захищені сторінки (доступ тільки для авторизованих).

## 3. Нотатки

- Clerk надає готові компоненти `<SignIn />` та `<SignUp />`, які ми вже використовуємо.
- Це економить близько 20 годин розробки порівняно з написанням Auth з нуля.
