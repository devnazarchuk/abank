# День 13: UI/UX, тестування та виправлення помилок

**Дата:** [[2026-02-03]]
**Мета:** Покращити зовнішній вигляд та зручність використання платформи, провести тестування всього функціоналу та виправити знайдені помилки.

---

## 1. Завдання на день

- [ ] **Покращення UI/UX:**
  - [ ] Пройтися по всьому шляху користувача (від реєстрації до оплати та перегляду курсу) і знайти місця, які є незручними або незрозумілими.
  - [ ] **Єдиний стиль:** Перевірити, що всі кнопки, форми, шрифти та кольори відповідають єдиному стилю.
  - [ ] **Адаптивність:** Перевірити, як сайт виглядає на мобільних пристроях та планшетах. Виправити основні проблеми з версткою.
  - [ ] **Сповіщення:** Додати спливаючі сповіщення для ключових дій (напр., "Курс успішно створено", "Помилка при завантаженні даних").
  - [ ] **Лоадери та заглушки:** Додати індикатори завантаження там, де дані завантажуються асинхронно, та інформативні заглушки для порожніх списків.
- [ ] **Функціональне тестування:**
  - [ ] Скласти чек-лист для тестування основного функціоналу.
  - [ ] **Сценарій 1 (Новий користувач):** Реєстрація -> спроба переглянути платну лекцію -> перехід на сторінку оплати -> (симуляція) оплата -> отримання доступу.
  - [ ] **Сценарій 2 (Адміністратор):** Вхід -> створення курсу з YouTube плейлиста -> додавання тесту до лекції.
  - [ ] **Сценарій 3 (Спільнота):** Додавання коментаря -> відповідь на коментар -> відправка приватного повідомлення.
- [ ] **Виправлення помилок (Bug Fixing):**
  - [ ] Створити список знайдених багів.
  - [ ] Визначити пріоритетність (від критичних до незначних).
  - [ ] Сфокусуватися на виправленні критичних та серйозних помилок, які блокують функціонал.

## 2. Результат дня

- Покращено візуальний вигляд та зручність платформи.
- Проведено повне тестування основного функціоналу.
- Виправлено найбільш критичні помилки.
- Платформа готова до фінального огляду перед деплойментом.

## 3. Нотатки

- На цьому етапі не потрібно досягати ідеального дизайну, але важливо, щоб платформою було приємно та легко користуватися.
- Тестування на реальних мобільних пристроях (а не тільки в емуляторі браузера) може виявити несподівані проблеми.
- Залучення до тестування сторонньої людини (напр., Андрія) може дати дуже цінний фідбек, оскільки він побачить продукт "свіжим поглядом".
