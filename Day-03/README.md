# День 3: Міграція на Open Source та налаштування інфраструктури

**Дата:** 25.01.2026
**Мета:** Розгорнути обрану Open Source платформу (AI-LMS), налаштувати локальне середовище (Docker, PostgreSQL) та підготувати проект до розробки.

---

## 1. Виконані завдання

- [x] **Вибір та клонування платформи:**
  - Як базу обрано проект **[akshsgaur/AI-LMS](https://github.com/akshsgaur/AI-LMS)**.
  - Основні переваги: Next.js 13+, інтеграція з Gemini AI, сучасний стек.
  - Проект клоновано в папку `Project-Code`.

- [x] **Налаштування локальної інфраструктури (Docker):**
  - Створено `docker-compose.yml` для запуску PostgreSQL.
  - Це дозволяє вести розробку безкоштовно, без залежності від хмарних провайдерів (Neon/AWS) на етапі тестування.

- [x] **Безкоштовний AI (Gemini):**
  - Налаштовано використання моделі `gemini-1.5-flash-latest`.
  - Це найшвидша та безкоштовна версія для розробників.

- [x] **Налаштування коду:**
  - Змінено драйвер бази даних з `neon-http` на стандартний `node-postgres` (pg).
  - Створено файл `.env.local` для локальних змінних.

## 2. Результат дня

- **[Project-Code](../Project-Code)**: Повністю робоча база коду.
- **Інфраструктура**: База даних запускається однією командою `docker-compose up`.
- **Економія**: Проект не потребує платних підписок для старту.

## 3. Як перевірити роботу

1.  Перейти в папку коду: `cd Project-Code`
2.  Запустити базу: `docker-compose up -d`
3.  Накотити схему БД: `npm run db:push`
4.  Запустити додаток: `npm run dev`
